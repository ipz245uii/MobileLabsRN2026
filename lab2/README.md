## Опис
Мобільний додаток на React Native з вкладеною навігацією та оптимізованими списками.

## Інструкція запуску
```bash
npm install
npx expo start
```
Відскануй QR-код у Expo Go.

## Реалізований функціонал

### Навігація
- Stack Navigator — перехід Новини → Деталі новини
- Bottom Tab Navigator — перемикання між Новини/Контакти
- Передача параметрів між екранами (назва новини у заголовку)

### FlatList (екран Новини)
- Pull-to-Refresh — оновлення списку жестом
- Infinite Scroll — підвантаження нових новин при прокрутці
- ListHeaderComponent — заголовок списку
- ListFooterComponent — індикатор завантаження
- ItemSeparatorComponent — розділювач між елементами
- Оптимізація: initialNumToRender=5, maxToRenderPerBatch=5, windowSize=10

### SectionList (екран Контакти)
- Групування контактів по секціях (Викладачі, Студенти, Адміністрація)
- renderSectionHeader — заголовки секцій
- Аватари з першою літерою імені
- ItemSeparatorComponent

## Висновки

**1. Чим відрізняється FlatList від ScrollView?**
FlatList використовує віртуалізацію — відображає лише видимі елементи, що економить пам'ять. ScrollView рендерить всі елементи одразу.

**2. Що таке віртуалізація списків?**
Механізм при якому відображаються тільки видимі елементи списку, а решта видаляється з пам'яті та відтворюється при прокрутці.

**3. Як здійснюється передача параметрів між екранами?**
Через navigation.navigate('Screen', { params }) та route.params на екрані-отримувачі.

**4. Що таке вкладена навігація?**
Коли один навігатор містить інший. Наприклад, Stack Navigator всередині Tab Navigator.

**5. У яких випадках застосовується SectionList?**
Коли дані потрібно відобразити групами з заголовками секцій, наприклад список контактів по категоріях.
